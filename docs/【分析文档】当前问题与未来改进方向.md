# 分析文档：当前问题与未来改进方向

> 日期：2026-05-09
> 基于 seed 7, 13, 42, 99 的模拟运行观察

---

## 一、当前状态评估

经过上一轮改进（开放空间带球、无球跑位采样、防守盯人/拦截、传球人/接球人状态），模拟已经初步呈现足球比赛的雏形：球员有进攻方向感，会左右盘带找空当，会尝试直塞和提前量传球，防守方会压迫和盯人。

但仍存在以下问题：

---

## 二、问题 1：前锋在门将前选择不佳

**现象**：部分前锋在接近球门时仍会向边路偏移，在射门角度变小后选择强行射门而非传给位置更好的队友。

**根因分析**：

- `choose_dribble_target()` 中射程内的目标偏向是渐进式过渡的（线性缩放 defender_weight 和 forward_bias），在射程边缘（~10-13m 处）防守人避让权重仍有约 20-60%，可能导致前锋绕开 GK 而非直冲球门
- `score_best_pass()` 只考虑传球线路是否被挡住和接球人是否空位，不考虑"接球人的射门角度是否理想"
- 当 dribble 得分低于 pass 得分时 AI 才会传球——但如果 dribble 仍得分更高（因为前方"有空间"），AI 会继续带球

**改进方向**：

1. **传球评分增加"接球人射门角度"权重**（用户提议）：对每个传球候选，计算接球人若直接射门的质量（距球门距离 + 角度），射门角度越好的队友传球得分越高。这会促使球员在禁区内选择横传而非小角度强射。

2. **射程内目标偏向曲线调整**：将线性过渡改为指数过渡（如 `goal_proximity²`），让靠近球门时更激进地直冲。在 6m 以内几乎完全忽略防守人。

3. **禁区边线惩罚加重**：在射程内，边线惩罚应与距球门距离联动——越靠近球门，越不能接受向边路移动。

---

## 三、问题 2：后卫站位过于激进

**现象**：防守球员尤其是 ANCHOR 压得太靠上，对方反击时身后大片空当。

**根因分析**：

- `defending_target()` 中非压迫防守球员现在会前压盯人/拦截线路，但没有"防守深度"约束
- `role_home_position(attacking=False)` 给出的防守基准位置对 ANCHOR 是 x=7（或 pitch_width-7），即距底线仅 7m——这是合理的低位。但拦截/盯人逻辑会让球员从基准位置前移去跟人
- 缺少"越位线"或"最后一道防线"的概念——当对方前锋已压过防线时，后卫应该优先回追而不是继续前压盯人

**改进方向**：

1. **防守深度约束**：非压迫球员在确定盯人/拦截位置后，与己方球门的距离不能小于一个下限（如距底线不低于 8m）。如果盯人目标已经深入，改为回追到目标与本方球门之间的位置。

2. **ANCHOR 特殊行为**：ANCHOR 作为最后一道防线，应始终保持在所有对方非持球球员与己方球门之间。除非对方 ANCHOR/PIVOT 已经进入禁区。

3. **反击检测**：如果对方控球时球已经越过中场线且本方防线身后有对方球员，触发"回追"模式——所有非压迫球员优先回撤而非前压盯人。

4. **防线整体性**：防守球员的纵向位置应参考最靠后的队友位置，避免防线脱节。

## 四、问题 3：射门/扑救机制需要精细化

**现象**：角度理想的单刀球经常不进，射门转化率偏低。

**根因分析**（[simulation.py:152-185](python_sim/simulation.py#L152-L185) 和 [simulation.py:387-416](python_sim/simulation.py#L387-L416)）：

当前射门模型：
- 射门目标 = 远门柱角 + `rng.uniform(-error_margin, error_margin)`
- `error_margin = (max(0.15, 1 - shot_quality/100) + facing_gap * 0.7) * 3.0`
- 球速固定 = `config.shot_speed` (18 m/s)

当前扑救模型：
- `save_chance = clamp(0.35 + save_quality/200, 0.35, 0.88)`
- 如果 `rng.random() > save_chance` → 不扑救

**问题**：
- 射门不区分球速和角度质量——软绵绵的地滚球和大力抽射判定相同
- 扑救只看门将数值，不考虑射门角度、球速、门将站位
- `save_chance` 下限 35% 意味着即使门将能力很差，面对 perfect shot 也有 35% 概率扑出——不合理
- 射门角度决定论：一个射向死角的球和一个射向中路的球，扑救概率相同

**改进方向**：

1. **射门质量三维度**：
   - **球速**：基于 `shot_speed` + 球员 `shooting` 能力。球速越快，GK 反应时间越短。
   - **角度**：目标点偏离 GK 站位越远，越难扑救。计算 `abs(aim_y - half_h) / (goal_width/2)` 作为角度质量。
   - **距离**：距球门越近，GK 反应时间越短。
   - **策略**：之前设定的“打死角”（见 【已完成】【阶段性技术文档3】202605091135修改日志.md 中的描述）不符合实际，应当调整为“根据门将的位置和球门的位置，判断打球门的哪一侧，更大概率打向门将不在的那一侧”

2. **扑救概率公式改进**：
   ```
   reaction_time = distance_to_ball / shot_speed
   gk_reach = save_quality / 100 * goal_width / 2  # GK 能覆盖的最大横向距离
   angle_difficulty = abs(aim_y - gk_y) / gk_reach
   save_chance = clamp(0.1 + save_quality/250 - angle_difficulty*0.4, 0.05, 0.85)
   ```
   - 低角度难度（射向 GK 站位附近）→ 高扑救率
   - 高角度难度（射向死角）→ 低扑救率
   - 近距离高速射门 → 低扑救率

3. **GK 站位优化**：当前 GK 在 `defending_target()` 中已随球移动，但站位逻辑较简单。可以让 GK 更积极地封堵近角。

## 五、问题 4：门柱/横梁击中

**现象**：当前没有门柱/横梁机制，射门不是进就是被扑/偏出。

**改进方向**（用户提议 + 细化）：

1. **门柱判定**：在 `_check_goal()` 中，球进入球门范围（y 在 goal_min~goal_max 之间，x 超出底线），但 y 非常接近 goal_min 或 goal_max（如 ±0.3m 以内）时，以一定概率（~15-25%）判定为击中门柱。
2. **横梁判定**：在 2D 模拟中不存在 z 轴，可以简化为：球进入球门范围但被判定为"击中横梁"。概率约 8-12%。
3. **反弹行为**：击中门柱/横梁后，球以降低的速度（原速度的 70-90%）沿反弹方向弹出。反弹角度为入射角的镜像 + 小量随机偏移。
4. **后续处理**：反弹球成为自由球（owner=None），任何球员可以争抢。这会产生"补射"机会。

## 六、改进优先级建议

| 优先级 | 问题 | 理由 |
|---|---|---|
| **高** | 射门/扑救机制（问题 3） | 直接影响比分和比赛结果可信度，改动范围小（仅 simulation.py） |
| **高** | 门柱机制（问题 4） | 增加戏剧性和真实感，改动范围小 |
| **中** | 传球评分增加射门角度（问题 1） | 改善进攻决策，减少不合理射门 |
| **中** | 后卫站位（问题 2 的防守深度约束） | 减少反击失球，让比分更合理 |
| **低** | 防线整体性（问题 2 其余） | 更复杂的改动，需谨慎设计 |

---

## 七、长期架构思考

当前 AI 框架（分层决策：阶段判断 → 动作选择 → 目标选择）是合理的。所有改进都是在这个框架内增加评分维度和约束条件，不需要架构性重构。

自对弈/ML 目前仍不推荐：
- 5v5 的状态空间虽然比 11v11 小很多，但训练一个有意义的策略网络仍需大量计算资源
- 当前启发式 AI 已经能产生合理的足球行为，继续细化评分函数是最快的改进路径
- 如果未来引入 ML，可以在现有评分函数基础上做参数优化（如用进化策略调优权重），而非端到端学习

建议的长期路径：**启发式 AI 打磨 → 参数自动调优 → (可选) 轻量级 RL 微调特定行为**
