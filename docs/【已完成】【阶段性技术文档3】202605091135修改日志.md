# 202605091135 修改日志

本次修改主要针对前期 Python 2D 验证版中暴露出的“连续进球不重置”、“防守乱跑扎堆”、“开球第一帧不可见”等现象进行了核心机制层面的修复与优化。

## 1. 修复进球重置与连续进球问题

**修改文件**：`python_sim/simulation.py`

**问题描述**：此前进球后没有真正的死球重置流程，导致出现短时间内连续进球的 bug。
**解决思路与实现**：
- **引入标准死球重置（Dead Ball Reset）**：修改了 `_reset_after_goal` 方法，进球后不再直接把球强行塞给开球方球员脚下，而是将球放置在球场正中心 `(pitch_width / 2, pitch_height / 2)`。
- **状态重置**：触发 `match.dead_ball = True`，记录 `restart_reason = "Kickoff"`，并重置所有球员的锁定跑位意图。
- **死锁悖论处理**：特殊修复了由 AI 判断带来的无限挂机（Deadlock）报错，统一死球延迟定格为2.0秒，恢复正常比赛节奏。
- **完善开球逻辑**：在 `_handle_restart` 中增加了中圈开球逻辑，系统会自动寻找开球方离球最近的球员切入并开球。

## 2. 约束门将位置与排斥扎堆 (空间占位意识)

**修改文件**：`python_sim/ai.py`

**问题描述**：守门员容易满场乱跑，同队球员也很容易扎堆挤在同一个区域（蜂群效应）。
**解决思路与实现**：
- **门将AI位置拦截增强（Angle Cutting）**：门将防守评估时会动态计算球中心与球门的“绝对连线”。现在门防能够针对远射，主动拉出一定的射击封堵线。但在无人员持有球且距离近于对手时，允许弃门扑救。
- **队友排斥机制（Separation Logic）**：计算球员跑位和回撤目标时，系统会扫描周边3米范围内的非门将队友，计算反向让出距离。

## 3. 初始第一帧开球与球可见性问题

**修改文件**：`python_sim/sample_data.py`

**问题描述**：刚打开对局或刚开球的瞬间，可视化器上没有显示球体。
**解决思路与实现**：
- **死球状态开局**：将初始状态设为 `dead_ball = True` 且 `restart_reason = "Kickoff"`，球刷在正中央。第一帧视觉效果完全可见。

## 4. 瞄准死角与传球遮挡判断

**修改文件**：`python_sim/ai.py`, `python_sim/simulation.py`

**问题描述**：射手总是无脑对着门将（门线中心处）踢；且传球线路经常无视路中间的防守者。
**解决思路与实现**：
- **引入死角瞄准偏差**：在 `_execute_shot` 与推演评分中，引入“打远角（Far Post）”锁定点，彻底废除默认居中瞄准。射术较低时会有倍率放大的射偏几率。
- **传球防遮挡检测（Line Intersection）**：新增 `distance_to_segment` 数学判定。传球将彻底扫描目标连线，如果该线段上存在防守人的防守/抄球半径，施加成倍的路径污染惩罚（Lane penalty）以逼迫转移球权方向。

## 5. 修复底线带球僵死问题（Deadlock at Baseline）

**修改文件**：`python_sim/ai.py`

**问题描述**：测试中发现当球员带球到了对方半场的极限底线时，他会原地卡住，随后全场所有的有球与无球球员均陷入死锁僵立。
**解决思路与实现**：
- **底线带球分压机制**：原本的 `score_dribble` 拥有“越接近球门得分越高”的正面收益，导致贴在底线（距离极近）时，AI始终认为“继续带球得分最高”，从而无限尝试往前带但受制于球场边界无法寸进。现在增加了距离检查（dist_to_goal < 2.0时收益为-10），逼迫球员必须传中或者射门倒三角，而不能无限日墙。
- **防守分离机制排他性补充**：原先由于上文增加的“避免队友扎堆排斥机制（Separation Logic）”，导致协防两名球员靠得太近时一直在推挤甚至不再上前逼抢抢断。此番更新后防守方在评估防线动作前，如果自己是“最近的关键逼抢人（Presser）”，将无视队友太近的排斥直接去抢脚下球。彻底解决了双方站街的僵局。
## 6. 死球状态逻辑升级与无球接应路线（Support & Restart）

**修改文件**：python_sim/ai.py, python_sim/simulation.py

**问题描述**：进球后排成斜线未回本方半场；发定位球/界外球时由于死球重置时间过短导致接应人跑不到位，且进攻时队友找空当的接应意识不足。
**解决思路与实现**：
- **开球回归己方半场**：废弃了之前硬编码让球员强制排成一排斜线的错误逻辑。在进球后的 Kickoff 环节，所有人会基于其战术主位置且受到半场边界限制，自动跑回各自的半场进行阵型落位；但发球手被豁免，站在中线负责开球。
- **动态接应意识（Open Passing Lane Search）**：这不仅解决了界外球没人来拿球的问题，更解决了活球时的进攻灵性。当己方获得球权时，所有无球队友在向进攻位置移动时会**实时检查他和持球人之间的空当连线**。一旦发现直线上有对方防守队员，他们会自动向四周（如向左向右拉开 3~5米）跳出防守阴影，主动暴漏在持球队友视野内接应。
- **调整死球延迟**：把定位球与界外球的发球等待时间从 2.0s 延长为 3.0s，开球延长至 4.0s，确保队友们能在这段保护时间内跑到接应的视点，展现出更好的定位球前压战术。
## 7. 开球限制与持球推进跑位优化（Kickoff constraint & Inward dribbling）

**修改文件**：python_sim/ai.py, python_sim/simulation.py

**问题描述**：中圈开球时开球人可以直接选择带球而不是传球；死球重启时间不足导致退防不到位；前插球员面对门将时经常选择一条无脑的底线带球路线并形成零度角打门。
**解决思路与实现**：
- **开球规则（Kickoff Force Pass）**：AI 评估球权动作时，如果识别到了当下的比赛源由于 Kickoff 而起且目前没有发生第一脚操作（last_touch_action None），那么发球队员无论如何计算分数，都只会被强制锁定出球。如果系统计算的 score_best_pass 仍未满则硬性给身边的随机队友出球。
- **归位时间延长**：所有类别的死球发球动作冷却延迟，现已由原来的 3s/4s 全局扩展到了 **10.0 秒**。使得所有的落位人员均能够充分复位回到防守框架与阵型基准点。
- **向球门内切（Inward Drift Dribbling）**：此前，持有球下底的边路球员在 score_dribble 打分很高时，系统为其给定的移动矢量只是无脑延续横坐标 x 向前推进（直线趟底边）。在此次设定中，当判定该球员位于前场危险区域（dist_x < config.pitch_width * 0.4），他的带球向量会自动沿中路目标即门将位置，实现收口“内切”。不仅彻底制止了他们抱着角旗跑到底线出界的弱智行为，同时也从根本上解决了带到零角度再射门的诡异行为。
