## 二、传球系统改进（上一轮）

### 2.1 可变球速 (Variable Pass Speed)

**问题**：所有传球速度固定为 `12.0 m/s`，与球员能力、距离、战术情境无关。

**修改**：

- **config.py** — 新增 9 个球速参数：

```python
pass_speed_min: float = 7.0          # 最低球速
pass_speed_max: float = 17.0         # 最高球速（受能力限制）
pass_speed_distance_per_m: float = 0.4   # 距离每增加 1m，速度 +0.4 m/s
pass_speed_type_through_bonus: float = 2.0  # 直塞球额外加速
pass_speed_type_lead_bonus: float = 1.0     # 提前量传球额外加速
pass_speed_pressure_malus: float = 2.5      # 受压时加速系数
```

- **ai.py** — 新增 `choose_pass_speed()` 函数（[ai.py:268](python_sim/ai.py#L268)）：

```python
def choose_pass_speed(player, target, pass_type, nearest_defender_dist, config) -> float:
    dist = distance(player.state.x, player.state.y, target.state.x, target.state.y)
    speed = config.pass_speed_min + dist * config.pass_speed_distance_per_m
    if pass_type == PassType.THROUGH_PASS:
        speed += config.pass_speed_type_through_bonus
    elif pass_type == PassType.LEAD_PASS:
        speed += config.pass_speed_type_lead_bonus
    if nearest_defender_dist < 3.0:
        speed += (3.0 - nearest_defender_dist) * config.pass_speed_pressure_malus
    quality = player.derived.pass_quality / 100.0
    max_speed = config.pass_speed_min + (config.pass_speed_max - config.pass_speed_min) * quality
    return clamp(speed, config.pass_speed_min, max_speed)
```

- **models.py** — `Intent` 新增字段（[models.py:80](python_sim/models.py#L80)）：
```python
pass_speed: float = 0.0  # 0 表示使用 config 默认值
```

- **ai.py** — `decide_ball_owner()` 中 4 处 Intent 构造点均加入 `choose_pass_speed()` 调用。

- **simulation.py** — `_execute_pass()` 从 `intent.pass_speed` 读取球速，若为 0 则回退到 `config.pass_speed`。

### 2.2 动态提前量 (Dynamic Lead Time)

**问题**：LEAD_PASS 和 THROUGH_PASS 的提前量硬编码为 0.55s 和 0.85s，不随飞行时间、接球人速度、防守人位置动态调整。

**修改**：

- **config.py** — 新增 6 个参数：

```python
lead_time_min: float = 0.25              # 最小提前量
lead_time_flight_fraction: float = 0.85  # 飞行时间因子
lead_time_receiver_speed_factor: float = 0.06  # 接球人速度因子
lead_time_through_extra: float = 0.4     # 直塞额外提前量
lead_defender_nudge_radius: float = 2.8  # 防守人避让半径
lead_defender_nudge_strength: float = 1.2  # 防守人避让强度
```

- **simulation.py** — 重写 `_resolve_pass_target()`（[simulation.py:562](python_sim/simulation.py#L562)）：
  - 动态 lead_time = `lead_time_min + flight_time * flight_fraction + receiver_speed * speed_factor`
  - 直塞再额外增加 `lead_time_through_extra`
  - 直塞前向偏移与传球距离成正比（`min(3.5, dist * 0.25)`），使用 `attack_direction` 而非硬编码 team_id
  - 预测点超出场地 2m 时回退为 TO_FEET（即传到接球人当前位置）

- **simulation.py** — 新增 `_nudge_away_from_defenders()`（[simulation.py:602](python_sim/simulation.py#L602)）：
  - 在目标点周围采样 9 个偏移候选
  - 对每个候选计算防守人距离惩罚（越近扣分越多）
  - 选取防守压力最小的点

### 2.3 高斯误差模型 (Gaussian Error Model)

**问题**：误差使用 `rng.uniform(-1.2, 1.2)` 均匀分布，产生"显著偏差"的球比例偏高，不符合现实中大多数球接近目标的规律。

**修改**：

- **config.py** — 新增 5 个参数：

```python
pass_error_base: float = 0.12          # 基础误差标准差
pass_error_quality_factor: float = 0.85  # 能力因子
pass_error_distance_per_m: float = 0.014  # 距离因子
pass_error_speed_factor: float = 0.03   # 速度因子
pass_error_facing_factor: float = 0.4   # 朝向因子
```

- **simulation.py** — `_execute_pass()`（[simulation.py:117-132](python_sim/simulation.py#L117-L132)）：
  - 使用 Box-Muller 变换生成高斯分布误差（替代 `rng.uniform`）
  - error_std = base + (1-quality)*quality_factor + dist*dist_factor + speed*speed_factor + facing_gap*facing_factor
  - 最终落点 clamp 到 `[0.5, pitch-0.5]` 边界

**误差校准表**（2σ 范围约占 95% 概率）：

| 场景 | quality | dist | speed | facing_gap | error_std | ±2σ |
|---|---|---|---|---|---|---|
| 完美短传 | 1.0 | 3m | 8m/s | 0.0 | 0.16m | ±0.32m |
| 典型组织传球 | 0.72 | 8m | 10m/s | 0.1 | 0.55m | ±1.10m |
| 冒险直塞 | 0.72 | 15m | 14m/s | 0.3 | 0.82m | ±1.64m |
| GK 开大脚 | 0.55 | 12m | 15m/s | 0.8 | 1.18m | ±2.36m |

### 2.4 传球类型决策扩展

**问题**：`choose_pass_type()` 的 THROUGH_PASS 判定过于宽松，且不考虑防守压力。

**修改** — `choose_pass_type()`（[ai.py:251](python_sim/ai.py#L251)）：
- 签名新增 `nearest_defender_dist` 参数
- THROUGH_PASS 触发条件收紧：必须同时满足 PIVOT 角色 + forward_value > 3.0 + dist > 5.0 + 防守人距离 > 1.8 + POSSESSION_ATTACK 阶段
- LEAD_PASS 触发条件：forward_value > 2.0 或 lateral_gap > 4.0 或 dist > 10.0

## 三、细节修正（本轮）

### 3.1 死球重新开始的发球逻辑

**问题**：边线球、角球、球门球重新开始后，发球球员有时带球移动而非传球，或者将球传给对手。

**根因**：

1. `_setup_restart()` 没有重置 `match.ball.last_touch_action`。当上一次触球动作是射门（SHOOT）时，该值不为 None，导致后续强制传球的判断条件 `match.ball.last_touch_action is None` 失败。
2. `decide_ball_owner()` 中的强制传球只针对 Kickoff（`match.restart_reason == "Kickoff"`），Ball 的 Corner/ThrowIn/GoalKick 不受强制传球约束。

**修改**：

- [simulation.py:486](python_sim/simulation.py#L486) — `_setup_restart()` 增加一行：
```python
match.ball.last_touch_action = None
```

- [ai.py:187](python_sim/ai.py#L187) — 强制传球条件泛化：
```python
# 之前：if match.restart_reason == "Kickoff" and match.ball.last_touch_action is None:
# 之后：
if match.restart_reason and match.ball.last_touch_action is None:
```

- [ai.py:194-195](python_sim/ai.py#L194-L195) — 兜底传球目标改为选择**最近的队友**（之前取列表第一个队友）：
```python
mate = min([p for p in team.players if p.player_id != player.player_id],
           key=lambda p: distance(player.state.x, player.state.y, p.state.x, p.state.y))
```

### 3.2 传球提前量出界

**问题**：大量 LEAD_PASS 和 THROUGH_PASS 的预期接球点在场外或贴近边线，加上误差后实际落点出界。

**修改**：

- [simulation.py:131-132](python_sim/simulation.py#L131-L132) — `_execute_pass()` 中误差叠加后的最终落点 clamp 到 `[0.5, pitch-0.5]`。
- [simulation.py:589-593](python_sim/simulation.py#L589-L593) — `_resolve_pass_target()` 中预测点超出场地 2m 时回退为传脚下（TO_FEET）。
- [simulation.py:597-599](python_sim/simulation.py#L597-L599) — 落点 clamp margin 从 1.0m 增加到 1.5m。
- [simulation.py:611-612](python_sim/simulation.py#L611-L612) — `_nudge_away_from_defenders()` 候选点 clamp 同步更新到 1.5m。
- [simulation.py:585-586](python_sim/simulation.py#L585-L586) — 直塞前向偏移改用 `owner_team.attack_direction`（之前硬编码 `1 if owner.team_id == "A" else -1`）。

### 3.3 红色队伍（Team B）LEFT/RIGHT 位置颠倒与朝向旋转

**问题**：

1. Team B 的 LEFT 球员（进攻方左侧球员）初始位于 y=7（场地下方），RIGHT 位于 y=17（场地上方）。但 Team B 进攻方向向左（attack_direction=-1），朝向对方球门时"左侧"应对应高 y 值。位置与实际左右颠倒。
2. 开球时 Team B 的 ANCHOR、LEFT、RIGHT 三人会原地旋转 180°，面朝自家球门，因为 kickoff 逻辑让球员退回到防守 home position，而初始位置比 home position 更靠前，导致球员向右移动（朝自家球门方向）。

**根因**：

- `role_home_position()`（[ai.py:355](python_sim/ai.py#L355)）中 LEFT 的 y 坐标固定为 `half_h - 5.5`（低 y），RIGHT 固定为 `half_h + 5.5`（高 y）。这对 attack_direction=1 的队伍正确，对 attack_direction=-1 的队伍方向颠倒。
- 同样问题存在于 `structured_target()` 的进攻站位、`decide_restart_position()` 的角球站位中。
- 开球时 `decide_restart_position()` 将非发球球员目标设为 `role_home_position(attacking=False)`——即防守 home position。这导致球员从初始位置回退到更靠后的防守位置，移动方向朝向自家球门。

**修改**：

- [ai.py:345-352](python_sim/ai.py#L345-L352) — 新增 `_role_flank_sign()` 辅助函数：

```python
def _role_flank_sign(team: Team, role: Role) -> float:
    if role == Role.LEFT:
        return -team.attack_direction   # 进攻向右时左侧为低 y，进攻向左时左侧为高 y
    if role == Role.RIGHT:
        return team.attack_direction
    return 0.0
```

- [ai.py:369-370](python_sim/ai.py#L369-L370) — `role_home_position()` 中 LEFT/RIGHT 改为：
```python
return (mid_x, half_h + flank * 5.5)
```

- [ai.py:398-401](python_sim/ai.py#L398-L401) — `structured_target()` 中 LEFT/RIGHT 进攻站位使用 `flank * 7.0`。

- [ai.py:143-157](python_sim/ai.py#L143-L157) — `decide_restart_position()` 角球站位使用 `flank * 3.0`（进攻方）和 `flank * 2.0`（防守方）。

- [ai.py:127-133](python_sim/ai.py#L127-L133) — 开球时不再退到防守 home position，改为原地保持（clamp 到己方半场）：
```python
if team.attack_direction == 1:
    tx = clamp(player.state.x, 1.0, config.pitch_width / 2 - 2.0)
else:
    tx = clamp(player.state.x, config.pitch_width / 2 + 2.0, config.pitch_width - 1.0)
```

- [sample_data.py:68-69](python_sim/sample_data.py#L68-L69) — Team B 的 LEFT/RIGHT 初始 y 坐标互换：
```python
# 之前：B_L at y=half_h-5（低 y, 错误）, B_R at y=half_h+5（高 y, 错误）
# 之后：B_L at y=half_h+5（高 y, 正确）, B_R at y=half_h-5（低 y, 正确）
```

- [ai.py:117](python_sim/ai.py#L117) — 修复一个分离逻辑 bug：检测近距队友时，排除条件从 `player.player_id != match.ball.owner_player_id` 修正为 `p.player_id != match.ball.owner_player_id`（原条件检查的是决策球员本人而非遍历到的队友 p）。

### 3.4 进球后重新开球站位

**问题**：进球后 `_reset_after_goal()` 不清除球员速度且不传送球员回半场。失球方原地不动，进球方缓慢走回自己半场，导致重新开球时双方站位混乱。

**修改** — `_reset_after_goal()`（[simulation.py:308-325](python_sim/simulation.py#L308-L325)）：

- 清零所有球员的 `vx, vy`
- 将双方所有球员传送到 `role_home_position(attacking=False)`（防守 home position）
- 双方球员朝向均设为对方球门方向

```python
for team in match.teams:
    for player in team.players:
        ...
        player.state.vx = 0.0
        player.state.vy = 0.0
        home_x, home_y = role_home_position(team, player.role, self.config, attacking=False)
        player.state.x = home_x
        player.state.y = home_y
        player.state.facing_angle = 0.0 if team.attack_direction == 1 else math.pi
        player.state.target_facing_angle = player.state.facing_angle
```

- [simulation.py:7](python_sim/simulation.py#L7) — 导入列表中新增 `role_home_position`。

## 四、验证结果

多 seed（7, 42, 99, 777, 1234）测试通过：

| 指标 | 结果 |
|---|---|
| 传球完成率 | 80%–95%（不同 seed 有波动） |
| 传球类型分布 | TO_FEET / LEAD_PASS / THROUGH_PASS 三种均有出现 |
| 死球后第一动作 | 始终为 PASS（不再出现带球或射门） |
| 开球时球员朝向 | Team A 朝右（0），Team B 朝左（π），全时段稳定 |
| Team B LEFT/RIGHT 位置 | LEFT 在 y=17（高 y，场地上方），RIGHT 在 y=7（低 y），符合进攻方向为左时的左右定义 |
| 进球后重新开球 | 双方均在各自半场的 defensive home 位置，朝向对方球门 |
| 传球直接出界率 | 约 10%–17%，较修改前显著下降 |
