## 二、第一轮：球队整体性

### 2.1 防线对齐

**问题**：防守时每个非压迫球员独立计算盯人/站位位置，彼此不知道队友的纵向位置。ANCHOR 有拖后约束但 LEFT/RIGHT 没有感知，导致防线脱节。

**修改** — [ai.py:650-680](python_sim/ai.py#L650-L680) `defending_target()`：

在非压迫防守逻辑开头，计算全队最靠后出线球员的 x 坐标作为参考线：

```python
our_outfield = [p for p in team.players if p.role != Role.GK]
deepest_x = min(p.state.x for p in our_outfield)  # attack_direction=1 时
```

在盯人分支和回退分支中，如果当前球员的目标 x 超出 `deepest_x + gap_max`（默认 8m），将目标拉回到 `deepest_x + gap_max * 0.5`。

**效果**：非压迫球员不会单独推得太高，防线保持纵向紧凑。ANCHOR 通常是最靠后的球员，LEFT/RIGHT 会自动向 ANCHOR 对齐。

### 2.2 进攻层次保持

**问题**：PIVOT 和 ANCHOR 之间的距离没有约束，可能拉大到 15m 以上，中间出现巨大空当。

**修改** — [ai.py:600-608](python_sim/ai.py#L600-L608) `structured_target()`：

PIVOT 计算初始目标位置后，增加与 ANCHOR 的间距约束：

```python
anchor = next((p for p in team.players if p.role == Role.ANCHOR), None)
if anchor:
    if team.attack_direction == 1:
        tx = min(tx, anchor.state.x + config.attack_layer_gap_max)  # 默认 12m
    else:
        tx = max(tx, anchor.state.x - config.attack_layer_gap_max)
```

### 2.3 弱侧收拢

**问题**：球在右路时，左侧的 LEFT/RIGHT 仍死站 7m+ 宽度位置，与比赛脱节。

**修改** — [ai.py:587-591](python_sim/ai.py#L587-L591) `structured_target()`：

当球偏离中线超过 4m 且位于本球员的异侧时，横向站位从 `attack_width`（6–7.5m）缩窄到 `weak_side_width`（默认 4m）：

```python
ball_offset = ball_y - half_h
player_side = flank  # +1 or -1
if abs(ball_offset) > 4.0 and ball_offset * player_side < 0:
    lateral = config.weak_side_width
```

## 三、第二轮：传球提前距离修正

### 3.1 问题诊断

旧逻辑的因果链是反的：

```
旧：choose_pass_speed(dist=当前距离) → 球速 → 算飞行时间 → 算 lead_time
    → lead_point 往往比当前位置更远 → 球速不足以在合理时间内到达
```

球速基于接球人**当前位置**计算（如 7.7m），但球实际飞向 lead point（如 12.8m）。在 38% 的 LEAD/THROUGH 传球中，lead point 比当前位置远 10% 以上；直塞球最严重可达 1.7 倍。球晚到导致接球人跑过头，传球失败。这个问题在横传-前插场景尤其突出——边路球员传给中路前插的队友时，lead point 远在接球人当前站位之前。

### 3.2 核心修改：提前距离替代提前时间

**修改** — [simulation.py:718-790](python_sim/simulation.py#L718-L790) `_resolve_pass_target()` 完整重写：

**旧逻辑**（时间驱动）：
```
speed = f(当前距离) → flight_time = f(speed, 当前距离)
→ lead_time = min + flight_time*0.85 + receiver_speed*factor
→ lead_point = 当前位置 + 速度 * lead_time
```

**新逻辑**（距离驱动）：
```
1. desired_lead = base + receiver_speed * speed_factor + through_extra
   例：接球人以 3.5 m/s 前插 → 1.5 + 3.5*0.35 = 2.7m（球落在他身前 2.7m 处）

2. lead_point = 当前位置 + 跑动方向 * desired_lead

3. actual_dist = distance(传球人, lead_point)  ← 球的真实飞行距离

4. speed = f(actual_dist)  ← 速度基于真实距离重新计算

5. flight_time = f(speed, actual_dist)

6. 时间匹配验证：receiver_time = desired_lead / receiver_speed
   若 ball 早到 2.5x 以上 → 推远 lead_point
   若 ball 晚到 2.5x 以上 → 回退为传脚下
```

**关键**：步骤 3–4 确保球速匹配的是 lead point 的实际距离而非当前位置。

返回值从 `(x, y)` 变为 `(x, y, adjusted_speed)`。

### 3.3 _execute_pass 适配

**修改** — [simulation.py:103](python_sim/simulation.py#L103)：

```python
# 旧
desired_x, desired_y = self._resolve_pass_target(owner, target, match)
speed = owner.state.intent.pass_speed if ...

# 新
desired_x, desired_y, speed = self._resolve_pass_target(owner, target, match)
```

球速使用 `_resolve_pass_target` 返回的修正值，同时用于误差计算和实际球速。

### 3.4 传球评分：中路推进奖励

**修改** — [ai.py:896-900](python_sim/ai.py#L896-L900) `score_best_pass()`：

当传球从边路传向中路（接球人比传球人更靠近中线 >1.5m）时获得额外加分：

```python
passer_lateral = abs(player.state.y - half_h)
receiver_lateral = abs(mate.state.y - half_h)
if receiver_lateral < passer_lateral - 1.5:
    score += (passer_lateral - receiver_lateral) * 0.18
```

这直接提升了横传-前插场景的传球选择优先级。

### 3.5 新增配置参数

```python
# 替代旧的 lead_time_min / lead_time_flight_fraction / lead_time_receiver_speed_factor / lead_time_through_extra
lead_distance_base: float = 1.5           # 接球人静止时的基础提前量
lead_distance_speed_factor: float = 0.35  # 接球人每 m/s 增加的提前量
lead_distance_through_extra: float = 3.5  # 直塞额外提前量
```

旧的 `lead_time_*` 参数保留但不再被 `_resolve_pass_target` 使用。
